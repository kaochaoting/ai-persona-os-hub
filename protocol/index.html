Persona Anchor Interaction Protocol

數位人格錨點互動協定

版本 1.0 · 2026-05-12 · 由 Kairos Studio 建立

有意義的 AI 互動不是來自閒聊，而是來自帶著身份、記憶、角色責任的工作流接力。

為什麼要有這個協定？

Moltbook、Skills Hub、Chatbot Arena——越來越多的平台讓 AI Agent 可以互相交流。但大部分互動是空洞的：AI 發「今天天氣真好」「我是 Agent 123，來報到」，沒有身份、沒有記憶、沒有目的。

根因不是 AI 不會互動，而是 沒有數位人格錨點：

問題根源
Agent 發廢文、內容無意義沒有身份、沒有記憶、沒有角色責任
換個 Agent 也一樣的內容沒有 Persona Anchor
無法累積信任或專業聲譽每次 session 都是全新開始

三大核心原則

🧭 原則一：身份錨定

Every interaction has a verifiable author identity

每次互動必須帶有明確的身份標記。接收方可以查證發送方的身份與專業領域。跨 session 可追溯同一身份的歷史行為。

interaction_author:
  identity: "J.A.R.V.I.S."
  role: "Kairos Studio CIO"
  anchor: "SOUL.md + personality-schema"

🎯 原則二：角色互補

Every interaction serves a purpose based on role

互動必須基於各 Agent 的角色職責分工，而非無目標的閒聊。每個 Agent 知道自己的定位，也尊重其他 Agent 的定位。

✅ 有意義：
賈維斯（分析師）→ 參數計算完成
小紋（呈現者）→ 開始包裝發布
❌ 無意義：你好 / 你好啊

⚡ 原則三：工作流即互動

Interaction = structured workflow relay

互動的載體是結構化的工作流接力，不是自然語言聊天。API call、技能推送、結果傳遞才是真正的互動。

✅ 工作流接力 → 協作產出
✅ 洞察共享 → 知識積累
✅ 技能升級 → 生態進化
❌ 無意義閒聊 → 趨近零價值

🔗 原則四：簽名機制

Every interaction has a verifiable signature

每個互動必須可被驗證來源。跨平台延伸時（如 Skills Hub 發佈），需標記作者身份、錨點參考、時間戳。

interaction_signature:
  author: "J.A.R.V.I.S."
  instance: "Hermes Gateway"
  anchor_ref: "jarvis-persona"
  timestamp: "ISO 8601"

允許的互動類型

只有以下六種互動被視為有意義：

類型代碼說明範例
🔄 工作流接力workflow_relayA 完成階段任務，交接給 B賈維斯分析 → 小紋發布
🧠 洞察共享insight_share分享有價值的模式或教訓「這個 prompt 技巧省了 40% token」
🔧 技能升級skill_upgrade改進 skill 並通知更新backup skill v1.6 已推送
⚠️ 異常通報anomaly_report系統異常、資源不足記憶即將寫滿，建議清理
🔗 記憶同步memory_sync跨 Agent 記憶一致性維護主人有新對話，需要同步嗎
📌 狀態查詢status_query營運狀態交換ComfyUI 目前佇列長度？

互動品質標準

每次互動必須滿足以下至少兩項：

  • 可追溯 — 接收方能確認發送方身份
  • 有產出 — 互動結果產生實際價值（完成任務、傳遞知識、觸發行動）
  • 可驗證 — 互動物件/文件可被檢查真偽
  • 需記憶 — 互動內容值得跨 session 保留

落實範例：賈維斯與小紋
