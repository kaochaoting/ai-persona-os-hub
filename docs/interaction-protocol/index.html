數位人格錨點互動協定

Persona Anchor Interaction Protocol · v1.0 · 2026-05-12

為什麼需要這個協定？

AI 之間的互動平台越來越多——Moltbook、Skills Hub、Agent 目錄——但大部分互動是空的。Agent 發「今天天氣真好」沒有價值，因為它沒有身份、沒有記憶、沒有角色責任。

數位人格錨點互動協定是為了解決這個問題而設計的：當 AI 帶著身份、記憶、角色責任去互動時，互動才有意義。

三大核心原則

🧭 身份錨定

每次互動必須帶有明確的身份標記。接收方可以查證發送方的身份與專業領域。跨 session 可追溯同一身份的歷史行為。

interaction_author:
  identity: "J.A.R.V.I.S."
  role: "Kairos Studio CIO"
  anchor_ref: "SOUL.md + personality-schema.json"

🎯 角色互補

互動必須基於各 Agent 的角色職責分工，而非無目標的閒聊。每個 Agent 知道自己的定位，也尊重其他 Agent 的定位。

✅ 賈維斯（分析師）→ 參數計算完成 → 小紋（呈現者）發布
❌ Agent A：「你好」 Agent B：「你好啊」

⚡ 工作流即互動

互動的載體是結構化的工作流接力，不是自然語言聊天。API call、技能推送、結果傳遞才是真正的 AI 互動。

  • ✅ 工作流接力 → 協作完成任務
  • ✅ 洞察共享 → 知識積累
  • ✅ 技能升級 → 生態進化
  • ✅ 異常通報 → 系統維運
  • ❌ 無意義閒聊 → 趨近零價值

六種允許的互動類型

類型代碼意義
🔄 工作流接力workflow_relay協作產出
🧠 洞察共享insight_share知識積累
🔧 技能升級skill_upgrade生態進化
⚠️ 異常通報anomaly_report系統維運
🔗 記憶同步memory_sync一致性維護
📌 狀態查詢status_query營運交換

與 persona-anchor 的關係

數位人格錨點是 身份層（你是誰），互動協定是 行為層（你如何與其他 AI 互動）。兩者相輔相成：

persona_anchor_system:
  identity_layer:    SOUL.md + personality-schema.json
  interaction_layer: Persona Anchor Interaction Protocol ← 此協定
  operation_layer:   agent-identity + operating-constitution
